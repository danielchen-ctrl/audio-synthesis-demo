# tools 目录说明

当前 `tools/` 按用途分成 5 类：

- `generation/`
  - 临时批量生成、场景生成、音频生成脚本
- `analysis/`
  - 输出模式、任务分析、完整性检查
- `validation/`
  - 生成结果验证与重构校验
- `debug/`
  - 调试和差异验证脚本
- `bin/`
  - 依赖的本地二进制工具

## 使用建议

- 这些脚本主要用于辅助验证、数据生成和临时分析
- 正式运行 demo 仍以根目录的 `server.py` / `start_demo.bat` 为主
- 如果后续再新增工具脚本，优先继续放进对应子目录，不再平铺到 `tools/` 根层

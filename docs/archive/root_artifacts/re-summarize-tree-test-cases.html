# 重新生成能力树状测试用例

## 来源说明

- PRD：`重新生成的功能升级`
- Figma：`Re-summarize / node 120:62787`
- 参考模板：[teams测试用例1.0.xlsx](C:\Users\PLAUD\Downloads\teams测试用例1.0.xlsx)
- 说明：本版本基于上一轮已成功读取到的 PRD 与 Figma 内容快照生成；当前飞书与 Figma MCP 临时不可用，因此以下用例对实时更新内容存在待确认项。

## 疑问点

1. Tab 层 `换语言` 在 PRD 中标记为 `P2` 且建议 MVP 不加，当前是否应完全隐藏入口，还是仅预留但不可点。
2. `分享链接中该 Note 消失` 的规则在 PRD 中标记为待确认，生成中到底是显示空态、隐藏当前 note，还是显示处理中提示，需要产品最终拍板。
3. `Ask Note` 是否允许走 `换个模板` 重新生成流程，历史记录中有待确认痕迹，当前先按“不支持”设计用例。
4. 发言人数量输入范围写的是“同提交转写时的输入”，但最小值、最大值、是否允许空值与非数字没有在当前材料里明确。
5. 第二次修改开始“扣时长”的扣减口径、额度不足时的具体文案与跳转策略，在当前 PRD 中未完全展开。

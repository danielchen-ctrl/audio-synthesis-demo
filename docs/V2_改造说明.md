## 一、改造背景

同事基于 FastAPI + Vue 3 + MySQL + Celery + MinIO 搭建了平台骨架（V2），具备基本的任务提交、文件管理和用户认证能力，但以下核心功能尚未实现：

| 缺失功能 | 原版状态 |
|---------|---------|
| 真人音色管理（注册/删除） | ❌ 完全没有 |
| TTS 降级保护 | ❌ CosyVoice 失败直接崩任务 |
| 音频静音裁剪 | ❌ 头尾有数字静音 |
| LLM 输出质量保障 | ❌ 无后处理，无质量门禁 |
| Few-shot 训练数据 | ❌ `few_shot/` 目录为空 |
| UUID 认证 Bug | ❌ 所有需鉴权端点返回 500 |
| 一键启动脚本 | ❌ 手动起 4 个服务 |

本次改造将 V1 本地平台（`audio-synthesis-demo`）经过验证的核心功能逐步接入 V2，同时修复原版 Bug。

### P1：音频合成补齐

#### 1.1 静音裁剪（silenceremove）

CosyVoice 合成的 WAV 文件头部通常有约 50ms 的数字静音（~-90dB），拼接后听感有轻微"顿感"。

**修改文件**：`backend/app/services/audio.py` → `_ffmpeg_normalize()`

加入 ffmpeg `silenceremove` 滤镜：
- 阈值 `-65dB`：保守设定，只裁数字静音，不影响正常语音（> -40dB）
- `start_duration=0.05s`：防止裁掉极短起音
- `stop_duration=0.15s`：保留自然句尾停顿

#### 1.2 CosyVoice 失败自动降级（EdgeTTSProvider）

**新增文件**：`backend/app/providers/tts/edge_tts_provider.py`

原版 CosyVoice 失败直接抛异常，任务 FAILED。改造后：

```
CosyVoice 合成失败
  → EdgeTTSProvider.synthesize()（edge_tts 12语言）
  → 降级成功 → 任务 SUCCEEDED + error_message = "[TTS_WARN] 部分片段降级为 edge_tts"
  → 降级也失败 → 任务 FAILED
```

**关键实现**：Celery worker 是同步线程，没有 asyncio 事件循环，必须用 `asyncio.new_event_loop()` 创建独立循环，不能用 `get_running_loop()`。

**修改文件**：`backend/app/services/audio.py`

`synthesize_lines()` 函数签名变更：
```python
# 原版（3-tuple，无降级）
def synthesize_lines(tts, lines, voice_assignments, language, output_format) -> tuple[bytes, float, list[dict]]:

# 改造后（4-tuple，支持降级）
def synthesize_lines(tts, lines, voice_assignments, language, output_format,
                     fallback_tts=None) -> tuple[bytes, float, list[dict], bool]:
#                                                                          ^^^^
#                                                          degraded=True 表示发生过降级
```

**修改文件**：`backend/app/tasks/generation.py`

```python
# 新增注入 fallback_tts
from app.providers.tts.edge_tts_provider import EdgeTTSProvider
fallback_tts = EdgeTTSProvider() if settings.EDGE_TTS_FALLBACK_ON_FAILURE else None

audio_bytes, duration_sec, segments, degraded = synthesize_lines(
    tts=tts, ..., fallback_tts=fallback_tts
)
if degraded:
    task.error_message = "[TTS_WARN] 部分片段降级为 edge_tts"
```

**新增 `.env` 字段**：
```dotenv
EDGE_TTS_ENABLED=true
EDGE_TTS_FALLBACK_ON_FAILURE=true
```

#### 2.2 音色管理 REST API

**新增文件**：`backend/app/api/v1/voices.py`

**`POST /api/v1/voices` — 注册新音色（含强制 E2E 验证）**

注册流程：
```
1. 接收 multipart：audio(file) + name + language + gender + reference_text(可选)
2. 调 CosyVoice POST /v1/voices/create → 获取 voice_id
   失败 → 返回 502
3. 立即合成验证短句：
   zh → "你好，这是音色验证。"
   en → "Hello, this is a voice verification."
4. 验证成功 → 写 voice_catalog DB → 返回 201 + {verified: true}
5. 验证失败 → 不写 DB → 返回 422 + {verified: false, message: "参考音频质量不足..."}
```

**`GET /api/v1/voices` — 列出音色（从 DB 读，不再实时拉 CosyVoice）**

**`DELETE /api/v1/voices/{voice_id}` — 删除音色**
- 仅创建者可删除（User 模型无 is_admin，暂不支持管理员特权）
- `?delete_remote=1` 同步删除 CosyVoice 服务端；失败只 log 不阻断本地删除

**修改**：`backend/app/api/v1/meta.py` → `GET /meta/voices` 改为从 DB 读，DB 为空时回退 fallback 列表，前端 API client 无需改动。

### P3：LLM 生成质量保障

#### 3.1 质量门禁（quality_check.py）

**新增文件**：`backend/app/services/quality_check.py`

基于 22 场景云 LLM vs Bundle LLM 对比评估，保留 3 条对云 LLM 有效的规则：

| 规则代码 | 触发条件 | 说明 |
|---------|---------|------|
| `core_marker_artifact` | 对话含 `<<…>>` 标记 | 云 LLM 偶发，保留作观察 |
| `high_repetition_rate` | 唯一行率 < 60%（行数 > 5）| 内容高度重复 |
| `word_count_critical_short` | 实际字数 < 目标 30% | 内容严重不足 |

**不迁移的 7 条 Bundle 专属规则**：`language_mismatch`、`high_chinese_ratio`、`scenario_placeholder_artifact`、`chinese_role_name_leak` 等——云 LLM 22 场景评估中零出现这些问题。

触发时任务标记 FAILED，`error_code` 写入对应规则代码。

#### 3.2 精简后处理（postprocess.py）

**新增文件**：`backend/app/services/postprocess.py`

两步处理（3A 精简路线，不做 repair 遍）：

**步骤 1 — 关键词注入** `enforce_keywords_in_lines()`：
- 检查用户指定关键词是否出现在对话中
- 中文：将关键词以上下文化短句形式注入现有行（不新增行）
- 英文：追加"We should explicitly cover: ..."说明行
- 已包含则跳过

**步骤 2 — 中文说话人稳定化** `stabilize_dialogue_constraints()`：
- 防止幽灵 Speaker（1人任务不出现 Speaker 2）
- 防止说话人顺序混乱
- 非中文路径直接 early return，零副作用

**容错设计**：任何异常只记 warning，返回原始 lines，不阻断合成流程。

**新增文件**：`backend/app/services/multilingual_naturalness_lite.py`
- 从 V1 `multilingual_naturalness.py`（2200行）移植
- 替换两处 V1 专用导入：`LANGUAGE_ALIASES` 内联、`load_text_naturalness_rules` 桩函数
- 保留 V1 已验证的单人对话 bug fix：`max(1, people_count)` 而非 `max(2, people_count)`

#### 3.3 接入 generation.py

**修改文件**：`backend/app/tasks/generation.py`，在 LLM 文本生成后、TTS 合成前插入：

```python
if mode == "llm" and settings.LLM_POSTPROCESS_ENABLED:
    target_words = max(50, int(params.get("target_duration_sec", 60) * 2.5))
    lines = apply_postprocess(
        lines, language=language,
        keywords=params.get("keywords", []),
        speaker_count=speaker_count,

        scenario=params.get("template", ""),

```

## 五、Few-shot 训练数据的作用

训练数据（933 条样本，位于 `backend/app/data/few_shot/`）**不是用来训练 DeepSeek 模型**，而是在每次生成任务时作为**上下文示例注入 Prompt**，引导 DeepSeek 模仿目标风格。

**工作流程**：

```
用户提交任务（场景=医疗随访，语言=中文）
    ↓
few_shot.py 从 933 条样本中检索最匹配的 1-2 条
（按 场景×语言 过滤，优先字数接近的高分样本）
    ↓
dialogue.py 把样本插入 System Prompt：
    "以下是同场景的真实对话样本，参考其风格生成内容不同的新对话"
    ↓
DeepSeek 模仿样本风格输出结果
```

**有无 few-shot 的差异**：

| 维度 | 无 few-shot | 有 few-shot |
|------|------------|------------|
| 语言风格 | 书面腔、PPT 感 | 口语化、真实工作对话节奏 |
| 场景贴合 | 通用框架 | 深入行业专有语言（医疗用语、金融术语等）|

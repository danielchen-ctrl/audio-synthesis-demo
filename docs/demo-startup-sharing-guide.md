# Demo 启动、分享与排障指南

## 适用场景

这份说明用于帮助同事或外部演示对象快速完成：

- 一键启动 Demo
- 在本机访问 Web 页面
- 通过局域网分享给其他电脑使用
- 下载生成的文本与音频
- 修改文本后重新合成音频
- 遇到常见问题时快速定位

## 一键启动

推荐直接双击：

- `start_demo.bat`

它会自动完成：

1. 启动服务
2. 等待服务就绪
3. 自动打开浏览器

默认访问地址：

- `http://127.0.0.1:8899/`
- `http://localhost:8899/`

## 局域网分享

启动成功后，页面右上角会显示“局域网访问地址”。

例如：

- `http://192.168.1.11:8899/`

把这个地址发给同一局域网内的其他电脑，对方即可直接打开 Web 页面。

也可以手动访问：

- `http://127.0.0.1:8899/api/server_info`

这个接口会返回：

- 当前监听端口
- 本机可分享的访问地址
- 分享提示信息

## 生成、编辑、下载的正确使用方式

### 1. 生成文本

在页面中填写：

- 人物画像
- 场景说明
- 对话核心内容
- 人物数量
- 字数限制

然后点击：

- `生成场景对话文本`

### 2. 修改文本

生成结果会出现在页面底部的可编辑文本框中。

你可以直接修改文本内容，例如：

- 调整措辞
- 删除某一段
- 插入新的发言
- 改成你需要演示的版本

### 3. 继续合成音频

修改完成后，点击：

- `生成 mp3 音频`

系统会优先使用当前文本框中的内容，而不是最初生成的原始文本。

### 4. 下载文本与音频

页面上的按钮：

- `下载文本`
- `下载音频`

都支持局域网访问用户直接使用。

注意：

- `下载文本` 会先保存当前编辑框里的最新内容，再开始下载
- `下载音频` 如果当前文本已被修改、且音频还不是最新版本，会先重新生成音频，再下载

## 文件会保存到哪里

生成结果默认保存在：

- `demo/`

每次生成通常会创建一个按时间戳命名的子目录，里面可能包含：

- 文本文件
- `manifest.json`
- 音频文件（`.mp3` / `.wav`）
- `segments.json`
- `transcript.vtt`
- `tts_input_debug.txt`

## 常见问题排查

### 1. 双击 `start_demo.bat` 后页面打不开

先检查：

1. 是否有黑色服务窗口仍在运行
2. 浏览器是否能打开：
   - `http://127.0.0.1:8899/`
3. 本机是否安装了 Python

如果命令窗口提示：

- `Python was not found`

请先安装 Python 3.8+。

### 2. 别人的电脑打不开你分享的地址

优先检查：

1. 你和对方是否在同一局域网
2. 你发的是不是 `192.168.x.x` 这类局域网地址，而不是 `127.0.0.1`
3. Windows 防火墙是否拦截了 `8899` 端口

建议先在你自己的电脑上访问：

- `http://你的局域网IP:8899/`

如果你自己也打不开，通常就是防火墙或网卡地址问题。

### 3. 下载按钮点了没反应

先检查：

1. 是否已经先生成文本
2. 下载音频前是否已经成功生成音频
3. 页面是否有报错提示

如果文本被编辑过，下载音频时系统可能会先重新生成音频，这时需要稍等几秒。

### 4. 生成音频失败

这类问题通常与本机 TTS 或音频处理环境有关。

优先检查：

1. 网络是否可访问 edge-tts 所需服务
2. `bin/ffmpeg.exe` 是否存在
3. 当前文本是否为空
4. 当前生成目录里是否有 `tts_input_debug.txt`

如果存在：

- `tts_input_debug.txt`

可以优先查看它，确认送入 TTS 的文本内容是否正确。

### 5. 服务端口 8899 被占用

当前启动脚本会尝试先清理占用 `8899` 的旧进程，再启动新服务。

如果仍然冲突，可手动检查：

```powershell
netstat -ano | findstr :8899
```

然后结束对应 PID。

### 6. 构建包缺失，服务启动失败

当前 Demo 依赖本地构建产物：

- `build/demo_app/SceneDialogueDemo.exe`
- `build/DialogDemo/DialogDemo.pkg`

如果这两个文件缺失，服务将无法正常启动。

## 推荐对外说明方式

如果你要把 Demo 发给别人试用，建议同时发这几条说明：

1. 先双击 `start_demo.bat`
2. 打开页面右上角显示的局域网地址
3. 先生成文本，再按需修改，再生成音频
4. 下载按钮可以直接保存文本和音频
5. 若访问失败，优先检查防火墙和是否处于同一局域网

## 当前已知限制

1. 当前服务主实现是 `src/demo_app/embedded_server_main.py`
2. Demo 依赖本地 build bundle，不是完全独立的纯源码服务
3. 局域网分享依赖当前电脑持续开着服务窗口
